# Codyssey B1-1: ChatGPT에게 일을 제대로 시키는 법 배우기

## 프로젝트 개요

이 저장소는 Codyssey B1-1 미션 수행을 위한 업무 자동화 프롬프트 패키지입니다.

선택한 업무 자동화 과업은 Codyssey 미션 코치 (교육 미션 분석, 개념 설명, 자기주도학습 계획 생성 자동화)입니다. 교육 프로그램의 미션 안내문을 입력하면 AI가 과제 목표, 필수 산출물, 기능 요구사항, 학습 순서, 제출 체크리스트, 검증 포인트를 정리하도록 설계했습니다.

또한 이 과업은 단순한 과제 정리 도구가 아니라, 미션 안에 등장하는 낯선 개념을 학습자 수준에 맞게 풀어주는 개념 길라잡이 역할도 포함합니다. 개념 길라잡이는 미션 수행 흐름의 첫 단계에 배치하되, 필수 프롬프트 개수를 늘리는 항목이 아니라 학습자의 이해도를 맞추는 사전 학습 보조 기능으로 설계했습니다.

사용자는 왕초보, 초보자, 중급자, 전문가 중 자신의 수준을 선택할 수 있습니다. 왕초보는 도구명과 기본 용어부터 설명받고, 초보자는 기본 개념의 쉬운 정의, 미션에서 필요한 이유, 간단한 예시, 추가 학습 프롬프트까지 확인합니다. 중급자와 전문가는 긴 개념 설명 대신 미션 수행에 필요한 핵심 개념과 주의할 점만 먼저 확인합니다.

## 비교한 모델/환경

| 제품군 | 환경 |
| --- | --- |
| ChatGPT | 무료 GPT 5.5 기본, 유료 GPT 5.5 Pro |
| Claude | 무료 Sonnet 4.6, 유료 Opus 4.6 (네이토) |
| Gemini | 무료 Flash 3.5, 유료 Gemini Pro 3.1 |

최종 선정 모델은 ChatGPT 유료(GPT 5.5 Pro)입니다. 선정 이유는 동일 입력 미션을 가장 촘촘하게 분석했고, 제출물 템플릿과 실행 체크리스트를 가장 실용적으로 제공했기 때문입니다.

모델 비교는 위 6개 환경으로 수행했고, 10턴 이상 실행 로그 검증은 최종 선정 모델인 ChatGPT 유료(GPT 5.5 Pro)로 수행했습니다.

> 모델명과 사용 환경은 사용 당시 웹 UI와 사용자가 확인한 표시명을 기준으로 기록했습니다.

## 동일 입력 데이터

모델 비교에는 실제 B1-1 원문이 아니라 Codex와 함께 만든 가상 미션 AI 기반 개인 지식관리 시스템 구축을 사용했습니다.

이유는 실제 B1-1 미션은 이미 학습 과정에서 여러 차례 분석했기 때문에, 새로운 가상 미션을 사용해야 모델별 기본 분석 능력을 더 공정하게 비교할 수 있다고 판단했기 때문입니다.

## 주요 설계 포인트

- v1 프롬프트는 사람이 처음 AI에게 묻는 것처럼 간단하게 작성했습니다.
- v1 비교 결과에서 나온 한계를 바탕으로 v2 시스템 프롬프트를 개선했습니다.
- 최종 프롬프트 테스트 피드백을 바탕으로 v3 시스템 프롬프트를 만들었습니다.
- 최종 유저 프롬프트 전문을 작성해 v3 시스템 프롬프트와 함께 재사용할 수 있게 했습니다.
- 학습자 수준을 왕초보/초보자/중급자/전문가로 설정해 답변 난이도와 예시 깊이를 조절할 수 있게 했습니다.
- 사용자가 처음부터 긴 미션 원문을 붙여넣지 않고, 학습 레벨을 먼저 선택한 뒤 AI가 미션 안내문 입력을 요청하도록 설계했습니다.
- 왕초보에게는 Google Form, Google Sheets, Slack 같은 도구명과 기본 용어부터 설명하도록 설계했습니다.
- 초보자에게는 미션 수행에 필요한 기본 개념을 설명하고, 개념 이해를 돕는 추가 학습 프롬프트를 제안하도록 설계했습니다.
- 긴 답변을 한 번에 제공하지 않고, 섹션별로 설명한 뒤 다음 단계로 넘어갈지 확인하도록 설계했습니다.
- 신뢰 가능한 공식 문서, 교육기관, 기술 블로그, 영상 자료를 우선 추천하도록 설계했습니다.
- 최종 프롬프트 실행 테스트에서 유튜브 추천 기능이 약하게 작동하는 것을 확인해, 한국어 사용자 기준의 검증된 유튜브 자료 또는 한국어 검색 키워드를 제안하도록 보강했습니다.
- 개념 길라잡이 기능은 미션 수행 흐름의 첫 단계에 배치하되, 필수 산출물이나 프롬프트 개수를 불필요하게 늘리지 않도록 제한했습니다.
- Few-shot 예시 3개 중 1개는 모호한 입력에 대해 AI가 확인 질문을 해야 하는 사례로 구성했습니다.
- 환각 검증 질문 5개를 준비해 원문에 없는 조건을 만들어내는지 확인하도록 설계했습니다.
- 실행 로그에는 10턴 이상 대화, 중간 조건 변경, 뒤늦은 추가 정보, 문맥 유지 검증을 포함했습니다.
