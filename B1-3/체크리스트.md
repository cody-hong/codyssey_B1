# B1-3 작업 및 제출 체크리스트

## 사용 방법

- 구현 전에 워크플로우 구조를 먼저 확정한다.
- 구현 중에는 각 도구별 구성 화면과 실행 결과를 반드시 캡처한다.
- 제출 전에는 민감정보가 보이지 않는지 다시 확인한다.

## 1. 프로젝트 1 공통 워크플로우 확정

- [ ] 비교할 도구 2개 이상 확정
- [ ] 동일하게 구현할 워크플로우 주제 확정
- [ ] Trigger 1개 이상 정의
- [ ] Action 2개 이상 정의
- [ ] 조건 분기(Filter/Router) 1개 이상 정의
- [ ] 각 분기 경로별 테스트 데이터 준비
- [ ] 성공/실패 판단 기준 정의

## 2. 프로젝트 1 구현

- [ ] 첫 번째 도구 계정/연동 준비
- [ ] 첫 번째 도구 워크플로우 구성
- [ ] 첫 번째 도구 조건 분기 설정
- [ ] 첫 번째 도구 실행 결과 확인
- [ ] 첫 번째 도구 구성 화면 캡처
- [ ] 첫 번째 도구 실행 결과 화면 캡처
- [ ] 두 번째 도구 계정/연동 준비
- [ ] 두 번째 도구 워크플로우 구성
- [ ] 두 번째 도구 조건 분기 설정
- [ ] 두 번째 도구 실행 결과 확인
- [ ] 두 번째 도구 구성 화면 캡처
- [ ] 두 번째 도구 실행 결과 화면 캡처
- [ ] 조건 분기별 결과가 각각 1회 이상 실행됐는지 확인

## 3. 프로젝트 1 비교 분석 보고서

- [ ] 사용한 도구 이름 작성
- [ ] 공통 워크플로우 구조 작성
- [ ] 각 도구 구현 과정 요약 작성
- [ ] 비교 항목 5개 이상 작성
- [ ] 각 도구 장점 정리
- [ ] 각 도구 단점 정리
- [ ] 어떤 상황에서 어떤 도구가 적합한지 의견 작성
- [ ] 유료 기능 사용 시 이유와 무료 대안 작성

## 4. 프로젝트 2 주제 확정

- [ ] 자동화할 반복 업무 1개 정의
- [ ] 현재 수작업 흐름 정리
- [ ] 자동화 후 기대 효과 작성
- [ ] 사용할 자동화 도구 1개 선정
- [ ] 선정 이유 작성
- [ ] Trigger 발생 시 자동 실행되는 구조인지 확인

## 5. 프로젝트 2 구현

- [ ] Trigger 1개 이상 포함
- [ ] Action 2개 이상 포함
- [ ] 조건 분기(Filter/Router) 1개 이상 포함
- [ ] 각 분기 경로 테스트
- [ ] 워크플로우 구성 화면 캡처
- [ ] 실행 결과 화면 캡처
- [ ] 워크플로우 흐름 설명 작성

## 6. 보너스 과제 선택 점검

- [ ] AI 연동 Action을 추가할지 결정
- [ ] AI 요약/분류/템플릿 메시지 중 사용할 기능 결정
- [ ] 실패 알림을 추가할지 결정
- [ ] 재시도 또는 대체 경로를 설계할지 결정

## 7. 보안 및 제출 전 점검

- [ ] API Key가 캡처나 문서에 노출되지 않았는지 확인
- [ ] 토큰이 캡처나 문서에 노출되지 않았는지 확인
- [ ] 비밀번호가 캡처나 문서에 노출되지 않았는지 확인
- [ ] 계정 이메일을 일부 가림 처리했는지 확인
- [ ] 공유 링크 권한이 과도하게 열려 있지 않은지 확인
- [ ] 실제 동작하는 워크플로우임을 실행 결과로 확인했는지 점검
- [ ] 프로젝트 1 보고서 파일이 있는지 확인
- [ ] 프로젝트 2 설계 문서 파일이 있는지 확인
- [ ] 각 도구별 구성 화면 캡처가 있는지 확인
- [ ] 각 도구별 실행 결과 캡처가 있는지 확인
