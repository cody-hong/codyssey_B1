# B1-3 노코드 자동화 미션 제작 Spec

## 목표

노코드 자동화 도구 미션을 제출 가능한 형태로 진행할 수 있도록 `B1-3` 폴더 안에 원문, 작업 안내, 체크리스트, 프로젝트별 보고서/설계 문서 틀을 만든다.

## 입력 자료

- 사용자가 제공한 미션 안내문
- 활성 작업 위치: `B1-3/`
- 기준 미션: "코딩 없이 마우스 클릭만으로 일이 자동으로 돌아가게 만들기"

## 작업 단계

1. 미션 원문을 `00. 미션 원문.md`로 정리한다.
2. `README.md`를 B1-3 노코드 자동화 미션 안내 문서로 갱신한다.
3. `handoff.md`에 현재 상태, 결정 사항, 다음 작업을 기록한다.
4. `체크리스트.md`에 구현/보고서/제출 전 점검 항목을 만든다.
5. 프로젝트 1 비교 구현 보고서 템플릿을 만든다.
6. 프로젝트 2 자유 주제 자동화 설계 문서 템플릿을 만든다.

## 산출물

- `00. 미션 원문.md`
- `README.md`
- `handoff.md`
- `체크리스트.md`
- `02. 프로젝트1_자동화도구_비교구현_보고서.md`
- `03. 프로젝트2_자유주제_자동화_설계문서.md`

## 검증 방법

- 미션 원문에 있는 필수 산출물과 기능 요구사항이 체크리스트에 빠짐없이 반영됐는지 확인한다.
- 프로젝트 1 문서에 2개 이상 도구 비교, 동일 워크플로우, 5개 이상 비교 항목이 들어갈 자리가 있는지 확인한다.
- 프로젝트 2 문서에 반복 업무 정의, 도구 선정 이유, 흐름 설명, 캡처/실행 결과 기록 자리가 있는지 확인한다.
- 보안 제약(API Key, 토큰, 비밀번호, 계정 이메일 마스킹)이 제출 전 점검에 포함됐는지 확인한다.

## 확인 질문

- 프로젝트 1에서 비교할 도구 후보는 Make와 Zapier로 시작해도 되는가?
- 프로젝트 1의 공통 워크플로우는 Google Forms 또는 Webhook 입력 -> 조건 분기 -> Google Sheets 기록 -> Slack/Discord 알림으로 진행해도 되는가?
- 프로젝트 2 자유 주제는 사용자의 실제 반복 업무를 받아 확정할 것인가, 아니면 예시 주제로 먼저 초안을 만들 것인가?
