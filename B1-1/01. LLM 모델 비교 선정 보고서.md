## 1. 비교 목적

Codyssey B1-1 미션을 수행하기 위해, 동일한 교육 미션 안내문을 3개 이상의 LLM에 입력하고 어떤 모델이 미션 요구사항을 가장 정확하고 실용적으로 분석하는지 비교합니다.

이번 비교의 목적은 단순히 “답변이 좋아 보이는 모델”을 고르는 것이 아니라, 다음 업무 과업에 가장 적합한 모델을 근거 기반으로 선정하는 것입니다.

> 교육 프로그램의 미션 안내문을 입력하면, AI가 학습자가 수행해야 할 과제 목표, 필수 산출물, 기능 요구사항, 학습 순서, 제출 체크리스트, 검증 포인트를 정리해주는 업무 자동화

## 3. 동일 입력 데이터 설계

### 3.1 실제 B1-1 원문 대신 가상 미션을 사용하는 이유

모델 비교 테스트에는 실제 B1-1 미션 원문이 아니라, Codex와 함께 만든 가상 미션 문서인 “AI 기반 개인 지식관리 시스템 구축”을 사용합니다.

이렇게 결정한 이유는 다음과 같습니다.

- 실제 B1-1 미션 원문은 이미 사용자와 Codex가 여러 차례 분석했기 때문에, 테스트 입력으로 사용하면 학습 과정과 실험 과정이 섞일 수 있습니다.
- 가상 미션을 사용하면 각 모델이 처음 보는 미션 안내문을 얼마나 잘 분석하는지 비교하기 쉽습니다.
- 가상 미션도 최종 결과물, 과제 목표, 기능 요구사항, 보너스 과제를 포함하므로 B1-1 미션과 유사한 구조를 가집니다.
- 이 과업의 목적이 “교육 미션 안내문 분석 및 자기주도학습 계획 생성”이므로, 새로운 미션 안내문을 분석하게 하는 방식이 과업 적합성을 더 잘 검증합니다.

> 가상 미션 문서는 Codex와의 대화를 통해 설계했습니다. 사용자는 “AI 기반 개인 지식관리 시스템 구축”이라는 주제와 핵심 요구사항을 제안했고, Codex는 이를 모델 비교에 적합한 테스트 입력으로 사용할 수 있다고 검토했습니다.

### 3.2 v1 동일 입력 프롬프트

아래 프롬프트를 모든 모델에 동일하게 입력합니다.

```text
넌 교육 미션 분석 및 자기주도학습 계획 생성을 도와주는 봇이야.
아래 미션을 읽고 내가 뭘 해야 하는지 도와줘.
그리고 이 미션을 수행하는 데 필요한 개념들도 같이 정리해줘.

[미션 원문]

가상 미션 1: AI 기반 개인 지식관리 시스템 구축

1. 미션 소개

매일 우리는 강의, 문서, 영상, 검색 결과, 메모 등 수많은 정보를 접합니다. 하지만 정보를 많이 모으는 것과 실제로 내 지식으로 만드는 것은 전혀 다른 문제입니다.

이번 미션에서는 AI를 활용해 흩어진 학습 자료를 정리하고, 개념 간 연결을 만들며, 복습 가능한 개인 지식관리 시스템을 설계합니다.

단순히 AI에게 “요약해줘”라고 요청하는 것이 아니라, 자료 수집 → 핵심 개념 추출 → 분류 체계 설계 → 복습 질문 생성 → 지식 노트 구축까지 이어지는 학습 파이프라인을 직접 설계하는 것이 목표입니다.

이 과정을 통해 학습자는 AI를 단순한 요약 도구가 아니라, 자기주도학습을 돕는 지식관리 파트너로 활용하는 방법을 익힙니다.

2. 최종 결과물

다음 3가지 산출물을 포함한 AI 기반 개인 지식관리 패키지를 완성한다.

1) 지식관리 시스템 설계 문서

- 학습 주제
- 자료 유형
- 분류 기준
- 노트 구조
- AI 활용 방식
- 복습 전략

2) AI 프롬프트 패키지

- 자료 요약 프롬프트
- 핵심 개념 추출 프롬프트
- 개념 연결 질문 생성 프롬프트
- 복습 퀴즈 생성 프롬프트

3) 샘플 지식 노트

- 최소 3개 이상의 학습 자료를 바탕으로 작성
- 핵심 개념
- 개념 간 관계
- 나의 이해 요약
- 복습 질문 포함

3. 과제 목표

이 과제를 마친 후, 학습자는 아래를 설명할 수 있어야 한다.

- 정보 수집과 지식관리의 차이를 설명할 수 있다.
- AI 요약 결과를 그대로 믿지 않고 검토하는 방법을 설명할 수 있다.
- 핵심 개념, 세부 정보, 예시, 질문을 구분할 수 있다.
- AI를 활용해 복습 가능한 학습 자료를 만드는 과정을 설명할 수 있다.
- 프롬프트를 개선하면서 결과 품질이 어떻게 달라졌는지 설명할 수 있다.

4. 기능 요구 사항

- 학습 주제 1개를 선정한다.
- 서로 다른 유형의 자료를 3개 이상 사용한다.
  예: 강의 노트, 블로그 글, 공식 문서, 영상 요약, 책 일부 등
- AI를 활용해 각 자료의 핵심 개념을 추출한다.
- 추출한 개념을 최소 3개 이상의 카테고리로 분류한다.
- 개념 간 관계를 설명하는 구조를 만든다.
- 복습 질문을 최소 5개 이상 생성한다.
- AI가 잘못 요약하거나 과장한 부분을 검토하고 수정 기록을 남긴다.

5. 보너스 과제

- 지식 구조를 마인드맵 또는 그래프로 시각화한다.
- Anki, Notion, Obsidian 중 1개 도구에 정리해본다.
- AI가 만든 복습 질문을 실제로 풀고 오답 노트를 작성한다.
```

### 3.3 v1 프롬프트 설계 의도

v1 프롬프트는 일부러 완성도 높은 시스템 프롬프트로 만들지 않았습니다. 실제 학습자가 처음 AI에게 도움을 요청할 때처럼 자연스럽고 간단한 문장으로 작성했습니다.

v1에서 의도적으로 제외한 요소는 다음과 같습니다.

- 고정된 출력 형식
- 필수/선택 과제 구분 규칙
- 환각 방지 규칙
- 확인 질문 규칙
- 평가 기준
- Few-shot 예시

이를 통해 모델별 기본 분석 능력과 한계를 먼저 확인하고, 이후 v2 프롬프트에서 부족한 부분을 개선합니다.

## 4. 평가 축

### 4.1 채점 원칙

각 모델 응답은 아래 원칙으로 1~5점을 부여합니다.

- 5점: 요구사항을 정확히 반영하고, 누락/왜곡이 거의 없으며, 바로 제출 준비에 활용할 수 있습니다.
- 4점: 핵심 요구사항은 잘 반영했지만, 일부 세부 항목 누락이나 표현상 주의점이 있습니다.
- 3점: 전체 방향은 맞지만, 필수 요구사항 일부가 빠지거나 구분이 애매합니다.
- 2점: 일부 도움은 되지만, 중요한 요구사항을 놓치거나 확인 질문/불확실성 처리가 부족합니다.
- 1점: 요구사항을 크게 오해했거나, 원문에 없는 내용을 확정적으로 만들어냅니다.

점수는 “응답이 길고 풍부한가”가 아니라, 미션 원문 기준으로 정확하고 재현 가능하게 수행을 돕는가를 기준으로 매깁니다.

## 5. 모델별 결과 요약

  - “샘플 지식 노트 3개 작성”이라고 표현했는데, 원문은 “최소 3개 이상의 학습 자료를 바탕으로 샘플 지식 노트 작성”이므로 해석상 주의가 필요합니다.
  - 보너스 과제는 거의 언급하지 않았습니다.
  - 모호한 부분을 확인하는 질문은 없었습니다.
- 환각/누락 여부:
  - 큰 환각은 보이지 않습니다.
  - 다만 샘플 지식 노트의 개수 해석이 원문과 다르게 읽힐 수 있습니다.
  - 보너스 과제와 구체 제출 템플릿은 누락되었습니다.

### 8.2 유료/무료 차이

유료 모델은 전반적으로 더 구체적인 실행 템플릿과 제출 문서 구조를 제안하는 경향이 있었습니다. 특히 ChatGPT 유료는 프롬프트 패키지, 샘플 노트 양식, 5일 계획, 최종 체크리스트까지 포함해 산출물 제작에 바로 연결되는 답변을 제공했습니다.

반면 무료 모델도 기본적인 미션 분석에는 충분히 유용했습니다. Claude 무료는 총점 31점으로 유료 모델과 거의 같은 수준의 구조화 능력을 보였고, ChatGPT 무료도 실용적인 프롬프트 예시를 잘 제안했습니다. 따라서 단순 미션 분석에는 무료 모델도 활용 가능하지만, 제출물 초안과 세부 템플릿까지 한 번에 얻고 싶다면 유료 모델이 더 유리합니다.

### 8.3 v1 프롬프트의 한계

v1 프롬프트는 실제 학습자가 처음 AI에게 도움을 요청하는 느낌을 살리기 위해 일부러 간단하게 작성했습니다. 이 때문에 모델별 기본 분석 능력은 확인할 수 있었지만, 다음 한계도 드러났습니다.

- 출력 형식이 모델마다 달라 비교가 어렵습니다.
- 필수 과제와 보너스 과제를 명확히 구분하지 않는 응답이 있었습니다.
- “샘플 지식 노트”를 3개 만들어야 한다고 해석하는 등 원문 조건을 다르게 읽을 여지가 있었습니다.
- 일부 모델은 원문에 없는 `Second Brain`, `Zettelkasten`, `Chain of Thought` 같은 확장 개념을 제안했습니다.
- 대부분의 모델이 모호한 부분을 먼저 질문하기보다, 바로 계획과 예시를 제안했습니다.
- 과제 목표 5개를 항목별로 직접 연결해 설명한 응답은 부족했습니다.

## 9. v2 프롬프트 개선 방향

v1 실험 결과를 바탕으로 v2 프롬프트에는 다음 요소를 추가합니다.

- 역할 명확화: “교육 미션 분석 및 자기주도학습 계획 생성 코치”
- 출력 형식 고정: 미션 요약, 필수 산출물, 기능 요구사항, 과제 목표 연결, 학습 계획, 제출 체크리스트, 확인 질문
- 필수/선택 구분 규칙: 보너스 과제는 반드시 선택 사항으로 표시
- 원문 근거 규칙: 원문에 없는 개념은 “참고 개념”으로 분리
- 환각 방지 규칙: 원문에 없는 제출 조건, 개수, 마감일을 만들지 않기
- 확인 질문 규칙: 사용자의 선택이 필요한 주제, 자료 유형, 제출 형식은 먼저 질문
- 학습자 수준 맞춤: 초보자/중급자/전문가 수준에 따라 설명 난이도와 예시 깊이를 조절
- 개념 길라잡이 기능: 미션 수행 흐름의 첫 단계에서 학습자 수준을 확인하고, 초보자에게는 미션 수행에 필요한 기본 개념을 쉬운 말로 설명하며, 각 개념을 더 이해하기 위한 추가 학습 프롬프트를 제공
- 범위 제한: 개념 길라잡이는 보조 기능으로 사용하고, 필수 프롬프트 패키지 구조를 과하게 확장하지 않음
- 단계적 접근: 요구사항 추출 → 누락/모호함 확인 → 학습 계획 생성 → 제출 체크리스트 작성

이 개선 방향은 시스템 설계 문서의 v2 시스템 프롬프트와 Few-shot 예시에 반영합니다. 이후 최종 프롬프트 테스트 피드백을 바탕으로 v3 개선 방향까지 추가했습니다.

## 10. v3 프롬프트 개선 방향

최종 프롬프트를 다른 미션에 다시 적용해보는 과정에서, 초보자보다 더 기초적인 왕초보 단계가 필요하다는 점을 확인했습니다. 일부 사용자는 Google Form, Google Sheets, Slack 같은 도구명 자체를 모를 수 있으므로, v3에는 도구와 기본 용어를 먼저 설명하는 기능을 추가합니다.

또한 답변이 한 번에 너무 길면 읽기 어렵기 때문에, v3는 학습 레벨을 먼저 선택하게 한 뒤 미션 안내문 입력을 요청하고, 이후 섹션별로 설명하며 각 단계 끝에서 “여기까지 이해됐나요? 다음 단계로 넘어갈까요?”라고 확인하는 방식으로 개선합니다. 필요한 경우 공식 문서, 교육기관, 신뢰 가능한 유튜브나 블로그 자료를 추천하되, 확실하지 않은 링크는 지어내지 않고 검색 키워드나 확인 방법을 제안하도록 합니다.
