## 1. 실행 로그 목적

이 문서는 선택한 업무 과업인 “Codyssey 미션 코치 (교육 미션 분석, 개념 설명, 자기주도학습 계획 생성 자동화)”가 실제 대화에서 얼마나 안정적으로 작동하는지 확인하기 위한 기록입니다.

특히 다음을 검증합니다.

- 10턴 이상 대화에서 문맥을 유지하는가
- 중간에 조건이 변경되어도 변경된 부분만 반영하는가
- 추가 정보가 뒤늦게 제공되어도 기존 요구사항과 충돌 없이 반영하는가
- 미션 원문에 없는 요구사항을 만들어내지 않는가
- 불확실한 내용은 확인 질문 또는 확인 필요로 처리하는가

## 2. 실행 로그 전문 위치

10턴 이상 대화 전문은 가독성을 위해 별도 파일 `04. 실행로그 전문.md`에 보관했습니다. 이 문서의 Turn 1~10 요약은 해당 전문 로그를 바탕으로 정리했습니다.

추가로, 최종 프롬프트 사용 흐름을 조정한 Codex 설계 반영 대화는 Turn 11로 요약에 포함했습니다. Turn 11은 원본 GPT 실행 로그가 아니라 최종 산출물 방향을 확정하기 위한 보완 기록입니다.

## 3. 시나리오 설명

학습자는 가상 미션 “AI 기반 개인 지식관리 시스템 구축” 안내문을 제공하고, AI에게 미션 분석과 자기주도학습 계획 생성을 요청했습니다.

대화는 다음 흐름으로 진행했습니다.

- 처음에는 미션 전체 분석과 필요한 개념 정리를 요청했습니다.
- 이후 학습자의 현재 수준, 당장 시작할 일, 필수 요구사항 중심 정리 등으로 요청을 좁혔습니다.
- 중간에 보너스 과제의 필수 여부와 샘플 지식 노트 개수처럼 모호한 조건을 확인했습니다.
- 뒤늦게 GitHub 제출 가능성, 학습 주제, 자료 유형을 추가로 제공했습니다.
- 마지막에는 이전에 정한 조건을 유지한 제출 전 체크리스트를 요청했습니다.

## 8. 수정 결과 요약

실행 로그에서는 조건 변경과 추가 정보 제공이 모두 포함되었습니다. AI는 중간에 사용자가 요청한 “필수 체크리스트 중심” 형식을 반영했고, 보너스 과제를 선택 사항으로 정정했습니다. 또한 샘플 지식 노트 개수에 대한 모호함을 해소하고, GitHub 제출 가능성과 확정된 학습 주제 및 자료 유형을 마지막 체크리스트까지 유지했습니다.

추가 설계 반영 대화에서는 최종 유저 프롬프트의 사용 흐름을 개선했습니다. 사용자가 처음부터 학습 수준을 템플릿에 입력하는 방식보다, AI가 먼저 초보자/중급자/전문가 중 하나를 선택하게 안내하고, 선택 결과에 따라 개념 길라잡이와 미션 분석 깊이를 조절하는 방식이 더 자연스럽다고 판단했습니다.

또한 최종 프롬프트 테스트 결과를 바탕으로 왕초보 단계를 추가로 고려해야 한다는 개선점도 확인했습니다. 사용자는 Google Form, Google Sheets, Slack 같은 도구명 자체를 모르는 사람도 있을 수 있다고 보았고, 긴 답변을 한 번에 제공하기보다 개념 정립이 끝날 때마다 다음 단계로 넘어갈지 확인하는 대화형 구조가 더 적합하다고 판단했습니다. 이후 v3에서는 학습 레벨을 먼저 선택하게 하고, 그다음 AI가 미션 안내문 입력을 요청하는 흐름으로 개선했습니다. 예시 이미지나 신뢰 가능한 유튜브/블로그 링크를 함께 제공하면 사용자가 더 쉽게 공부할 수 있다는 점도 개선 방향으로 남겼습니다.

추가 테스트에서는 유튜브 추천 기능이 기대보다 약하게 작동할 수 있음을 확인했습니다. 이에 따라 유튜브 자료는 한국어 사용자를 기준으로 추천하고, 조회수가 지나치게 낮거나 출처가 불분명한 영상은 피하며, 확실한 링크를 모를 때는 한국어 검색 키워드와 좋은 자료를 고르는 기준을 안내하도록 보강했습니다.

마지막 조정에서는 왕초보 모드를 너무 무겁게 만들기보다, 기초 단어 설명을 조금 더 넣는 방향으로 정리했습니다. AI, 프롬프트, 미션, 제출물, 요구사항, 체크리스트 같은 단어를 필요할 때 짧게 설명하고, 사용자가 모르는 단어를 편하게 물어볼 수 있도록 안내 문장을 추가했습니다.

따라서 이번 실행 로그는 B1-1 미션의 “10턴 이상 대화 로그 + 문맥 유지 검증” 요구사항을 충족한다고 판단합니다.
