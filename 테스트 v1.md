# 테스트 v1

## 목적

최종 프롬프트가 기존 가상 미션이 아닌 다른 미션에서도 잘 작동하는지 확인하기 위한 테스트 기록입니다.

이번 테스트에서는 다음을 확인합니다.

- AI가 먼저 학습 레벨(초보자 / 중급자 / 전문가) 선택을 안내하는가
- 학습자가 초보자를 선택했을 때 개념 길라잡이를 먼저 제공하는가
- 필수 산출물과 선택/보너스 과제를 구분하는가
- 미션 원문에 없는 조건을 임의로 만들지 않는가
- 모호한 부분을 확인 질문으로 남기는가
- 최종 출력이 미션 분석, 학습 계획, 제출 체크리스트로 이어지는가

## 테스트에 사용한 최종 프롬프트

```text
[미션 안내문]
아래에 내가 수행해야 하는 교육 프로그램 미션 원문을 붙여넣을게.
미션 원문을 기준으로 해야 할 일과 제출물을 정리해줘.

[요청]
1. 먼저 내 학습 레벨을 선택할 수 있게 안내해줘.
2. 내가 레벨을 선택하면 그 레벨에 맞춰 미션 분석을 진행해줘.
3. 내가 "초보자"라고 답하면, 먼저 개념 길라잡이를 제공해줘.
4. 내가 "중급자" 또는 "전문가"라고 답하면, 긴 개념 설명은 줄이고 미션 수행에 필요한 핵심 개념과 주의할 점만 먼저 정리해줘.
5. 학습 레벨 확인 후에는 지정한 출력 형식에 맞춰 미션을 분석해줘.
6. 원문에서 모호하거나 내가 선택해야 하는 부분은 임의로 정하지 말고 확인 질문으로 남겨줘.
7. 원문에 없는 제출 조건, 마감일, 도구 사용 의무는 만들지 말아줘.
8. 개념 길라잡이는 첫 단계에서 제공하되, 필수 산출물이나 필수 프롬프트 개수를 늘리지는 말아줘.
9. AI가 만든 분석 결과에서 오류, 과장, 누락이 생길 수 있는 부분과 수정 기록을 남기는 방법도 알려줘.

[출력 형식]
1. 미션 핵심 요약
2. 필수 산출물
3. 선택/보너스 과제
4. 기능 요구사항 체크리스트
5. 과제 목표 연결
6. 추천 진행 순서
7. 제출 전 검증 체크리스트
8. 확인 질문
9. 수준별 핵심 개념 설명
10. 개념 이해용 추가 학습 프롬프트
11. 참고 개념
12. AI 결과 검토 및 수정 기록 가이드
```

## 테스트 미션 원문

```text
분야
AI 도구 학습
구분
AI 도구
학습시간
40시간
코딩 없이 마우스 클릭만으로 일이 자동으로 돌아가게 만들기
문제기술
기술적 설명

1. 미션 소개
매일 아침 이메일을 확인하고 스프레드시트에 옮겨 적고 팀 채널에 요약을 공유하는 루틴을 떠올려 봅시다. 한 번에 5분이면 끝나는 일이지만 하루에 열 번 반복되면 어느새 50분이 사라집니다.

"이걸 자동으로 돌릴 수 없을까?" 하고 찾아보면 코드를 짜야 한다는 벽에 막히고 결국 다시 수작업으로 돌아간 경험이 있을 겁니다.

노코드 자동화 도구는 바로 이 지점에서 출발합니다.

Make나 Zapier 같은 플랫폼은 코딩 없이 Trigger(시작 이벤트)와 Action(처리 동작)을 시각적으로 연결해 반복 업무를 자동화합니다.

이번 미션에서는 동일한 워크플로우를 서로 다른 2개 이상의 자동화 도구로 구현하고 비교 분석한 뒤 자유 주제로 나만의 자동화 파이프라인을 설계·구현합니다.

단순히 "돌아가기만 하면 된다"가 아니라 Trigger와 Action의 작동 원리를 이해하고 조건 분기를 설계하며 도구별 장단점을 근거 있게 비교하는 것이 핵심입니다.

이 과정을 마치면 여러분은 반복 업무를 분석해 자동화 가능 여부를 판단하고 적합한 도구를 선정해 실제 워크플로우를 구축하는 역량을 갖추게 됩니다.

여기서 쌓은 노코드 자동화 설계 경험은 이후 AI 에이전트 연동이나 코드 기반 자동화로 확장해 나가는 데 탄탄한 발판이 될 것입니다.

2. 최종 결과물
다음 2가지 프로젝트 산출물을 완성한다.

[프로젝트 1] 자동화 도구 비교 구현
- 동일한 자동화 워크플로우를 2개 이상의 도구로 구현
- 각 도구별 워크플로우 구성 화면 캡처
- 실행 결과 화면 캡처
- 비교 분석 보고서 (Markdown 또는 PDF)

[프로젝트 2] 자유 주제 자동화 설계 및 구현
- 자동화할 반복 업무 1개 정의
- 자동화 도구 1개 선정 및 선정 이유 작성
- 워크플로우 설계 문서 (설명 또는 다이어그램 포함)
- 구현 화면 캡처
- 실행 결과 화면 캡처

3. 과제 목표
이 과제를 마친 후, 학습자는 아래를 스스로 설명할 수 있어야 한다.

- Trigger와 Action의 개념을 설명할 수 있다.
- 조건 분기(Filter/Router)의 역할을 설명할 수 있다.
- 서로 다른 자동화 도구의 특징을 비교할 수 있다.
- 특정 업무에 적합한 도구를 선택하고 그 이유를 설명할 수 있다.
- 자동화 흐름을 단계별로 설명할 수 있다.

4. 기능 요구 사항
다음 요구사항을 모두 만족해야 한다.

공통 요구 사항
- 실제로 동작하는 자동화 워크플로우를 구현해야 한다.
- Trigger 1개 이상 포함
- Action 2개 이상 포함
- 조건 분기(Filter/Router) 1개 이상 포함
- 조건 분기 구조가 포함된 경우 각 분기 경로가 실제로 1회 이상 실행된 결과를 확인할 수 있어야 한다.

프로젝트 1 요구 사항
- 서로 다른 2개 이상의 자동화 도구를 사용한다.
- 동일한 워크플로우 구조로 구현한다.
- 비교 분석 보고서에는 아래 항목을 포함한다.
  - 사용한 도구 이름
  - 구현 과정 요약
  - 최소 5개 이상의 비교 항목
  - 각 도구의 장단점 정리
  - 어떤 상황에서 적합한지 의견 작성

프로젝트 2 요구 사항
- 자동화할 반복 업무 1개를 정의한다.
- 도구 1개를 선정하고 선정 이유를 작성한다.
- 자동 실행 구조를 구현한다.
- 워크플로우 흐름 설명을 포함한다.

5. 보너스 과제 (선택)

보너스 1 – AI 연동 Action 추가
- 워크플로우에 생성형 AI(예: ChatGPT, Claude 등)를 Action으로 추가해 텍스트(요약/분류/템플릿 메시지)를 자동 생성한다.

보너스 2 – 실패 알림 및 재시도 전략
- 워크플로우 실행 중 오류가 발생하면 이메일/메신저로 실패 알림을 보낸다.
- 가능하다면 재시도(재실행) 또는 대체 경로(예: 저장 실패 시 임시 시트에 적재)를 1개 이상 설계한다.

6. 제약 사항

구현 및 동작 제약
- 자동화 툴을 비교 분석하기 위해 도구를 반드시 2개 이상 직접 사용해야 한다.
- 두 프로젝트 모두 "실제 동작하는 워크플로우"여야 한다(설계 문서만 제출 불가).
- 프로젝트 2는 Trigger 발생 시 자동 실행되어야 한다.

보안 및 제출물 제약
- 제출물(스크린샷/문서/공유 링크)에 API Key, 토큰, 비밀번호 등 민감정보가 노출되지 않도록 해야 한다.
- 필요 시 값은 마스킹 처리(예: ***)하고 계정 이메일도 일부 가림 처리한다.

계정/연동 환경 리스크 완화 가이드
- 권장 최소 구성 예시: Google Sheets + Gmail(또는 Email) + Webhook
- 권장 최소 구성 예시: Google Sheets + (Slack 또는 Discord 중 1개) + Webhook

과금 리스크 완화 가이드
- 무료 플랜으로 완수 가능한 조합을 우선 고려한다.
- 유료 결제가 필요한 기능을 사용했다면 "왜 불가피했는지"와 "무료 대안"을 보고서에 함께 정리한다.

7. 결과 예시
아래는 정답이 아니라 참고 예시다. 실제 내용은 달라도 된다.

프로젝트 1 예시

자동화 도구 비교 구현 (Make vs Zapier)

워크플로우: "Google Form 제출 → 조건 분기(점수 기준) → Google Sheets 기록 + Slack 알림"

[Make 구현]
- Trigger: Google Forms – Watch Responses
- Router: 점수 80점 이상 / 미만 분기
- Action 1: Google Sheets – Add a Row (합격자 시트)
- Action 2: Slack – Send a Message (합격 알림)

[Zapier 구현]
- Trigger: Google Forms – New Response in Spreadsheet
- Filter: 점수 80점 이상만 통과
- Action 1: Google Sheets – Create Spreadsheet Row
- Action 2: Slack – Send Channel Message

비교 항목 예시
1) UI/UX: Make은 시각적 노드 기반 / Zapier는 리스트 기반
2) 설정 난이도: Make은 초기 학습 곡선 있음 / Zapier는 직관적
3) 연동 서비스 범위: 두 도구 모두 광범위하나 세부 커넥터 차이 존재
4) 무료 플랜: Make 1,000 Ops/월 / Zapier 100 Tasks/월(예시)
5) 실행 로그: Make은 실행 히스토리에서 단계별 데이터 확인 가능 / Zapier는 Task History

프로젝트 2 예시

자유 주제: 팀 회의록 자동 정리 및 알림

반복 업무 정의: 매주 회의 후 회의록을 Google Docs에 작성하면 핵심 내용을 요약해 Slack 채널에 공유하는 작업

선정 도구: Make (시각적 분기 설계가 직관적이고 무료 Ops 범위 내 구현 가능)

워크플로우 흐름:
1) Trigger: Google Docs – Watch Documents (새 문서 감지)
2) Action 1: 텍스트 파싱(제목/참석자/결정사항 추출)
3) Filter: 결정사항이 존재하는 경우에만 다음 단계로
4) Action 2: Slack – Send a Message (요약 내용 + 담당자 멘션)
5) Action 3: Google Sheets – Add a Row (회의록 이력 기록)
```

## AI 응답 원문 2: 초보자 선택 후 응답

```text
초보자 레벨로 진행하겠습니다.

## 0. 초보자용 개념 길라잡이

### 1) 노코드 자동화 도구

**쉬운 정의**
코딩을 직접 하지 않고, 화면에서 앱들을 연결해 반복 업무가 자동으로 실행되게 만드는 도구입니다. 예: Make, Zapier 등.

**이 미션에서 필요한 이유**
이번 미션의 핵심은 “코딩 없이 마우스 클릭 중심으로 자동화 워크플로우를 만들 수 있는가”입니다. 따라서 노코드 자동화 도구를 직접 사용해 실제로 동작하는 흐름을 만들어야 합니다.

**간단한 예시**
Google Form에 새 응답이 들어오면 → Google Sheets에 기록하고 → Slack에 알림을 보내는 흐름입니다.

**추가 학습 프롬프트**
“노코드 자동화 도구가 무엇인지 초보자도 이해할 수 있게 예시 중심으로 설명해줘.”

---

### 2) Trigger

**쉬운 정의**
자동화가 시작되는 사건입니다. “무슨 일이 일어나면 자동화를 시작할까?”에 해당합니다.

**이 미션에서 필요한 이유**
기능 요구사항에 **Trigger 1개 이상 포함**이 명시되어 있습니다. 자동화는 Trigger 없이 스스로 시작될 수 없습니다.

**간단한 예시**
새 이메일이 도착함, Google Form 응답이 제출됨, 스프레드시트에 새 행이 추가됨, Webhook 요청이 들어옴.

**추가 학습 프롬프트**
“자동화에서 Trigger의 역할을 이메일, 스프레드시트, 메신저 예시로 설명해줘.”

---

### 3) Action

**쉬운 정의**
Trigger 이후 자동으로 수행되는 작업입니다. “자동화가 실제로 무엇을 할까?”에 해당합니다.

**이 미션에서 필요한 이유**
기능 요구사항에 **Action 2개 이상 포함**이 있습니다. 즉, 시작 이벤트만 있는 것이 아니라 최소 두 가지 작업이 이어져야 합니다.

**간단한 예시**
Google Sheets에 행 추가하기, Slack 메시지 보내기, 이메일 발송하기, 텍스트 요약하기.

**추가 학습 프롬프트**
“자동화에서 Action이 무엇인지 Trigger와 비교해서 쉽게 설명해줘.”

---

### 4) 조건 분기, Filter, Router

**쉬운 정의**
조건에 따라 자동화 흐름을 나누는 기능입니다. “이 조건이면 A로, 아니면 B로” 보내는 역할입니다.

**이 미션에서 필요한 이유**
기능 요구사항에 **조건 분기(Filter/Router) 1개 이상 포함**이 있습니다. 또한 분기 구조가 있다면 각 분기 경로가 실제로 1회 이상 실행된 결과를 확인할 수 있어야 합니다.

**간단한 예시**
점수가 80점 이상이면 “합격” 시트에 기록하고 Slack에 합격 알림을 보냄.
점수가 80점 미만이면 “불합격” 시트에 기록하고 다른 알림을 보냄.

**추가 학습 프롬프트**
“Filter와 Router의 차이를 Make와 Zapier 같은 노코드 자동화 도구 기준으로 설명해줘.”

---

### 5) 워크플로우

**쉬운 정의**
Trigger부터 Action, 조건 분기, 결과 저장까지 이어지는 전체 자동화 흐름입니다.

**이 미션에서 필요한 이유**
프로젝트 1에서는 같은 워크플로우를 2개 이상의 도구로 구현해야 하고, 프로젝트 2에서는 자유 주제로 나만의 워크플로우를 설계하고 구현해야 합니다.

**간단한 예시**
Google Form 제출 → 점수 조건 분기 → Google Sheets 기록 → Slack 알림.

**추가 학습 프롬프트**
“자동화 워크플로우를 설계할 때 Trigger, Action, 조건 분기를 어떤 순서로 정리하면 좋은지 알려줘.”

---

# 1. 미션 핵심 요약

이번 미션은 **코딩 없이 노코드 자동화 도구를 사용해 실제로 동작하는 자동화 워크플로우를 만들고, 도구별 차이를 비교하는 과제**입니다.

완성해야 하는 프로젝트는 2개입니다.

**프로젝트 1: 자동화 도구 비교 구현**
동일한 자동화 워크플로우를 **서로 다른 2개 이상의 자동화 도구**로 구현하고, 각 도구의 화면과 실행 결과를 캡처한 뒤 비교 분석 보고서를 작성합니다.

**프로젝트 2: 자유 주제 자동화 설계 및 구현**
자신이 정한 반복 업무 1개를 자동화 대상으로 정하고, 자동화 도구 1개를 선택해 실제로 동작하는 워크플로우를 구현합니다.

핵심은 단순히 “한 번 실행해봤다”가 아니라, 다음을 설명할 수 있어야 한다는 점입니다.

* 어떤 Trigger로 자동화가 시작되는지
* 어떤 Action들이 순서대로 실행되는지
* 조건 분기인 Filter 또는 Router가 어떤 역할을 하는지
* 왜 특정 도구를 선택했는지
* 도구별 장단점이 무엇인지
* 실행 결과가 실제로 확인되었는지

학습시간은 원문 기준 **40시간**입니다.

초보자 관점에서는 “이 도구가 좋다”라고만 쓰기보다, **내가 만든 자동화 흐름에서 왜 좋았는지**를 연결해서 쓰는 것이 중요합니다.

예를 들어 다음처럼 작성하면 좋습니다.

> Make는 Router를 시각적으로 확인하기 쉬워 조건 분기 구조를 이해하고 캡처하기 좋았다. 반면 Zapier는 단계별 설정이 직관적이어서 처음 자동화를 만드는 데 부담이 적었다.

# 6. 추천 진행 순서

## 1단계. 자동화 주제 후보 정하기

먼저 너무 복잡한 업무보다 간단한 반복 업무를 고르는 것이 좋습니다.

추천 후보는 다음과 같습니다.

| 후보        | 예시 흐름                                                 |
| --------- | ----------------------------------------------------- |
| 설문 응답 처리  | Google Form 제출 → 조건 분기 → Sheets 기록 → Slack/Discord 알림 |
| 이메일 분류    | 새 이메일 도착 → 키워드 조건 분기 → Sheets 기록 → 알림 전송              |
| 문의 접수 관리  | Webhook 또는 Form 접수 → 유형 분기 → 담당 채널 알림 → 기록            |
| 일정/회의록 정리 | 새 문서 생성 → 조건 확인 → 요약 또는 기록 → 알림                       |

초보자에게는 **Google Form 또는 Google Sheets 기반 자동화**가 비교적 이해하기 쉽습니다.

---

## 2단계. 프로젝트 1용 공통 워크플로우 설계

프로젝트 1은 같은 구조를 2개 이상의 도구로 만들어야 하므로, 먼저 공통 구조를 단순하게 정합니다.

예시 구조입니다.

> Google Form 제출
> → 점수 또는 카테고리 조건 분기
> → Google Sheets에 기록
> → Slack 또는 Discord에 알림

이때 필수 조건을 만족하려면 최소한 다음이 들어가야 합니다.

| 구성요소     | 예시                      |
| -------- | ----------------------- |
| Trigger  | Google Form 새 응답        |
| 조건 분기    | 점수 80점 이상 / 미만          |
| Action 1 | Google Sheets에 행 추가     |
| Action 2 | Slack 또는 Discord 메시지 전송 |

---

## 3단계. 프로젝트 1을 도구 2개 이상으로 구현

예를 들어 Make와 Zapier를 사용할 수 있지만, 원문에서 특정 도구를 의무로 지정하지는 않았습니다. 중요한 것은 **서로 다른 2개 이상의 자동화 도구를 직접 사용**하는 것입니다.

구현할 때는 각 도구에서 다음 화면을 캡처합니다.

| 캡처 대상         | 설명                                  |
| ------------- | ----------------------------------- |
| 전체 워크플로우 화면   | Trigger, Action, 조건 분기가 보이게 캡처      |
| Trigger 설정 화면 | 어떤 이벤트로 시작되는지 보이게 캡처                |
| 조건 분기 설정 화면   | 어떤 기준으로 나뉘는지 보이게 캡처                 |
| Action 설정 화면  | 어떤 작업이 실행되는지 보이게 캡처                 |
| 실행 결과 화면      | 실제 실행 성공 결과, 로그, 기록된 데이터, 발송된 메시지 등 |

---

## 4단계. 분기별 테스트 데이터 만들기

조건 분기가 있다면 각 경로가 최소 1회 이상 실행되어야 합니다.

예를 들어 점수 기준 분기라면 다음처럼 테스트합니다.

| 테스트 데이터 | 예상 분기     |
| ------- | --------- |
| 점수 90점  | 80점 이상 경로 |
| 점수 60점  | 80점 미만 경로 |

각각 실행 결과를 캡처해두면 제출 전 검증에 도움이 됩니다.

---

## 5단계. 프로젝트 1 비교 분석 보고서 작성

보고서에는 최소 5개 이상의 비교 항목이 있어야 합니다.

추천 비교 항목은 다음과 같습니다.

| 비교 항목     | 볼 포인트                                    |
| --------- | ---------------------------------------- |

# 10. 개념 이해용 추가 학습 프롬프트

## Trigger 이해용

> 노코드 자동화에서 Trigger가 무엇인지 초보자에게 설명해줘. Google Sheets, Gmail, Google Forms, Webhook 예시를 각각 들어줘.

## Action 이해용

> Trigger와 Action의 차이를 쉽게 설명해줘. 하나의 자동화 예시에서 Trigger 1개와 Action 2개를 구분해서 보여줘.

## Filter/Router 이해용

> 노코드 자동화에서 Filter와 Router의 차이를 설명해줘. 점수 80점 이상과 미만으로 나누는 예시를 들어줘.

## 워크플로우 설계 연습용

> “Google Form 제출 → 조건 분기 → Google Sheets 기록 → Slack 알림” 구조의 자동화 워크플로우를 초보자도 이해할 수 있게 단계별로 설계해줘.

## 비교 분석 보고서 작성용

> Make와 Zapier로 같은 자동화 워크플로우를 만들었다고 가정하고, UI/UX, 설정 난이도, 연동 서비스, 무료 플랜, 실행 로그 확인 방식 기준으로 비교 분석 보고서 초안을 작성해줘.

## 검증 체크리스트 작성용

> 노코드 자동화 과제를 제출하기 전에 확인해야 할 검증 체크리스트를 Trigger, Action, 조건 분기, 실행 결과, 민감정보 마스킹 기준으로 만들어줘.

## 오류 검토용

# 11. 참고 개념

## 1) Make와 Zapier

원문에서 예시로 언급된 대표적인 노코드 자동화 도구입니다.

* Make: 시각적인 노드 기반 흐름을 만들기 좋음
* Zapier: 단계별 리스트 방식으로 자동화를 만들기 좋음

다만 원문은 Make와 Zapier를 반드시 사용하라고 지정하지 않았습니다. **서로 다른 2개 이상의 자동화 도구를 직접 사용**하는 것이 필수입니다.

---

## 2) Webhook

Webhook은 특정 이벤트가 발생했을 때 다른 서비스로 데이터를 보내는 방식입니다.

초보자 관점에서는 이렇게 이해하면 됩니다.

> 어떤 서비스가 “새 데이터가 들어왔어!”라고 자동화 도구에 신호를 보내는 통로.

원문의 권장 최소 구성 예시에 Webhook이 포함되어 있습니다.

---

## 3) 실행 로그

실행 로그는 자동화가 실제로 실행되었는지, 어느 단계에서 성공하거나 실패했는지 보여주는 기록입니다.

이번 미션에서는 실행 결과 캡처가 필요하므로, 실행 로그 화면은 좋은 증빙 자료가 됩니다.

| 무료 플랜 범위         | 도구 정책이 바뀔 수 있음                      | 실제 도구의 현재 요금제 화면에서 확인      |
| 연동 서비스 범위        | 특정 앱 연동 가능 여부가 계정/플랜에 따라 다를 수 있음    | 직접 도구에서 커넥터 검색             |
| 실행 가능 여부         | AI가 그럴듯한 흐름을 제안해도 실제로는 설정이 안 될 수 있음 | 반드시 직접 실행 테스트              |
| Filter/Router 설명 | 도구마다 용어와 작동 방식이 다를 수 있음             | 실제 설정 화면 기준으로 설명           |
| 비교 항목            | 사용하지 않은 기능까지 경험한 것처럼 쓸 위험           | 직접 사용한 범위와 관찰한 내용만 작성      |
| 장단점              | 개인 경험이 아니라 일반론으로 과장될 수 있음           | “내가 구현한 워크플로우 기준”이라고 범위 명시 |
| 보안               | 캡처에 토큰, 이메일, Webhook URL이 남을 수 있음   | 제출 전 이미지 확대 확인             |

## 2) 과장 표현을 줄이는 방법

피해야 할 표현:

> Zapier는 모든 자동화에 가장 적합하다.
> Make는 무조건 Zapier보다 좋다.
> 이 도구는 오류가 전혀 없다.

권장 표현:

> 내가 구현한 Google Form 기반 워크플로우에서는 Zapier가 단계별 설정을 이해하기 쉬웠다.
> Make는 Router 구조를 시각적으로 확인하기 쉬워 조건 분기 설명에 유리했다.
> 무료 플랜 범위와 연동 앱 제한은 실제 계정 환경에 따라 달라질 수 있다.
```

## 개선 필요 사항

- 초보자 선택 후 개념 길라잡이가 먼저 제공되어 핵심 기능은 정상 작동함
- 응답이 매우 길기 때문에 실제 사용 시에는 사용자가 원하는 섹션부터 단계적으로 이어가게 하는 방식도 고려 가능
